수업 설계 핵심

학생이 "y = ax + b가 왜 직선인가"를 스스로 발견하게 한다. 교사가 먼저 설명하지 않는다.
도입 조작 → 전개 3단계 레이어 → 마무리 피드백 루프가 하나의 수업 자산으로 쌓인다.

일차함수 조작 카드 — a값(기울기)·b값(y절편)을 슬라이더로 조정하면 그래프가 실시간 변화

학생이 슬라이더를 움직이며 직선이 어떻게 바뀌는지 직접 확인한다. 교사는 설명 없이 관찰만 유도한다. 조작 카드는 비로그인 링크로 바로 접속 — 학생 가입·학운위 처리 불필요.

세 레이어를 모두 쓸 필요는 없다. 이번 차시에서 완성형만 써도 된다. 레이어는 교사가 선택하는 폭이지 의무 순서가 아니다.

AI가 교사를 대체하지 않는다. 반복적인 분류 작업을 자동화해서 교사가 핵심 판단에 집중하게 한다. 콴다 대비 차별점: 풀이 과정 기반 피드백.

투자적 설계 판단

교사가 쓴 시간이 "다음 수업 자산"으로 남는 구조인가? 학생의 서술 이력이 "내 학습 궤적"으로 보이는가?
이 수업 시나리오는 투자적 설계를 지향한다 — 데이터가 다음 수업의 출발점이 된다.

HEART 지표 체크
Happiness
긍정적으로 기억하는가?
Peak: AI 생성기 입력 후 즉시 쓸 수 있는 자료가 나왔을 때.
End: 대시보드에서 "기울기 혼동 9명" 확인하고 다음 발문이 떠오르는 순간.
현재 비바샘 End는 "자료 다운로드 후 로그아웃" — 감정이 없다.
Engagement
의미 있는 사용이 반복되는가?
수업 데이터 → 대시보드 확인 → 다음 자료 준비 루프가 설계되면 반복 접속이 생긴다. 현재는 루프가 없어서 이탈.
Adoption
첫 사용자가 핵심 가치를 경험하는가?
완성형 자료로 빠른 첫 경험 설계. 스마트PPT 그대로 쓰는 것만으로도 "쓸 만하다" 를 느끼게 한다.
Retention
다시 돌아오는 이유가 있는가?
현재: 학기 초 1회 방문 후 이탈. 이유: 데이터가 없으니 돌아올 이유 없음.
설계 목표: 피드백 루프 데이터 → 다음 차시 도입 설계로 연결. 데이터가 교사를 끌어당기는 루프.
Task Success
원하는 일을 끝까지 끝냈는가?
자료 검색 → 수정 → 배포 → 피드백 확인까지 비바샘 안에서 완결. 현재는 PC 다운로드 후 이탈 구조.
B = M × A × P — 행동 설계 점검
동기(M) — "Gemini 자비 결제하던 걸 비바샘에서 무료로"충분
능력(A) — 비로그인 정책 미해결 시 진입 자체 불가장벽 큼
자극(P) — 단원 시작 전 "AI 자료 생성해보기" CTA설계 필요
핵심 병목
능력(A)이 가장 낮다. 비로그인 정책 해결이 최우선 — 이게 풀려야 B(행동)가 발생한다.
EA Canvas — 경험 5층위
1
맥락 — 언제, 어디서, 어떤 상태에서?
▾

수업 일주일 전 ~ 전날 밤. 교사 혼자 노트북 앞. Gemini나 구글로 자료 찾다가 비바샘도 열어보는 상황. 학기 초에만 접속해본 교사.

2
행동 — 어떤 행동을 유도하는가?
▾

단원 검색 → 완성형 자료 확인 → AI 생성기 조건 입력 → 생성 결과 수정·저장 → 비로그인 링크 배포 → 수업 후 피드백 루프 확인.

3
시스템 — UI·Flow·알고리즘이 행동을 만드는가?
▾

비상 교과서 메타데이터 기반 검색 / AI 생성기 입력 UX / 비로그인 배포 링크 생성 / 피드백 루프 대시보드.

4
삶의 영향 — 365일 반복되면?
▾

수업 준비 시간 단축 + 학생 오류 패턴 누적 파악 + 비바샘이 수업 포트폴리오가 됨. 교사가 서비스 없이도 더 잘 가르칠 수 있게 되는 것이 목표.

5
설계 결정 — 투자적인가?
▾

투자적 설계. 교사가 쓸수록 다음 수업이 더 쉬워지는 구조. 학생 데이터가 다음 차시 출발점이 된다.

월요일 체크리스트 — 이 기획의 답

시작 맥락: 수업 준비 전날 밤, 교사 혼자 자료를 찾는 순간.
누적 가치: 수업 데이터가 쌓여 다음 차시 준비가 점점 빨라진다.
감정 잔여: 교사 — "대시보드 보니까 다음 수업이 보인다". 학생 — "내 오답이 정리돼서 왔다".
